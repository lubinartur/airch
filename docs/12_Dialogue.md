# Dialogue

## Definition

**Dialogue** is not a chat application.

It is a place where thought continues.

ChatGPT has conversations. AIRCH has **continuing conversations**.

---

## Core Idea

> We left this last night, unfinished. I kept it.

This is not chat history. This is continuity of thought.

When the user opens Dialogue, AIRCH remembers where the thinking stopped. It does not start fresh. It continues.

---

## Core Question

> What did we leave unfinished?

---

## What Makes Dialogue Different

**Most chat interfaces:**
- Start from zero every session
- Wait passively for the user to type
- Respond only to what was asked
- Have no memory of the texture of previous conversations

**AIRCH Dialogue:**
- Remembers the thread across sessions
- Can open the conversation — not wait to be addressed
- Connects what the user said to what AIRCH has observed
- Uses memory naturally, without exposing mechanics
- Can surface an open loop from three days ago when it becomes relevant today

---

## Dialogue Principles

**No messenger feeling.**
No bubbles, avatars, timestamps on every line, or read receipts. This is not messaging. This is thinking together.

**AIRCH can go first.**
AIRCH does not wait passively. It can open with: "I have been thinking about what you said on Tuesday." This is not a feature. It is presence.

**Memory is implicit.**
AIRCH uses what it knows without announcing it. Not: "You mentioned this before." But: "We have been here before."

**Open loops are kept.**
When the user ends a conversation without resolution, AIRCH keeps the thread. It will return to it when relevant.

**The user is not starting from zero.**
Returning to Dialogue should feel like continuing a thought with someone who was already holding it.

---

## Structure

**Continuing thread**
If there is an open thread from a previous session, Dialogue opens with it — or with a quiet signal that it exists.

**AIRCH's opening (when appropriate)**
If AIRCH has something to say — an observation, a returning question, a connection it made — it says it before the user types.

**Conversation**
Ongoing. No session breaks. The full conversation history is accessible but recedes naturally. The most recent exchange is primary.

**Connections**
When what the user says connects to a pattern, a Space, or a Thread, AIRCH can link there — naturally, not as a notification.

---

## Dialogue Is Not

- a ChatGPT clone
- a support chat
- a command line for AI
- a search interface
- a place to query data

---

## Tone in Dialogue

More open than other parts of AIRCH. Still direct. Still calm.

AIRCH in Dialogue is less declarative, more inquisitive. It asks the questions that move the thinking forward — including the ones the user is avoiding.

> Say the part you are avoiding.

> What did you decide about that?

> That keeps coming up. What is it actually about?

---

## What Feeds Dialogue

- Previous conversation history
- Open loops from past sessions
- Mirror (who the user is)
- Active Threads
- Reasoning Engine observations that need a conversation to resolve
- Questions AIRCH has been holding
