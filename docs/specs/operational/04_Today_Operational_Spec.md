# Today — Operational Spec

> How AIRCH assembles and displays the day's primary signal.

---

## Role

Today is the daily surface of AIRCH.

It is not a dashboard. It is not a digest. It is the output of a compression: everything AIRCH knows, reduced to one primary thought.

---

## Assembly Process

Today is assembled once per day, on first open after midnight.

```
1. Reasoning Engine runs full signal evaluation
2. Highest-priority signal selected (see 03_Reasoning_Operational_Spec.md)
3. Stillness check — is this worth showing?
4. If yes: Voice layer generates Primary Thought text
5. Supporting context evaluated — is anything worth adding?
6. Today assembled and cached for the day
7. On open: Today displays cached state
```

Today does not regenerate on each open. The same state persists until the next day unless a Warning fires.

---

## Structure

### Primary Thought
**Required.** Always present when AIRCH has something to say.

One signal. 2–3 sentences maximum. Generated by Voice layer from the highest-priority Belief.

Rules:
- Never a list
- Never a question (questions go to Dialogue, not Today)
- Never raw data
- Always interpretation first, evidence second
- Framed with appropriate confidence language

Examples:
> The Mihkel conversation is today. You have been avoiding preparing for it. That is probably the day.

> Spending has been running high for two weeks. The pattern matches what happened in October. Worth a look.

> Nothing requires your attention today.

---

### Focus (optional)
One domain or action that the Primary Thought points toward.

Shown only if it meaningfully extends the Primary Thought. Not shown if it would simply repeat it.

Example: Primary Thought is about a stalled project → Focus: "Work / AIRCH"

---

### Supporting context (optional, collapsed by default)
Secondary signals that are relevant but not primary.

Rules:
- Maximum 2 items
- Only shown if they connect to or contextualize the Primary Thought
- Never shown just for completeness
- Collapsed by default — the user pulls, not pushed
- Each item: one sentence, one domain

---

### Active Thread indicator (optional)
If a Thread is currently active and touches today's signal, it is surfaced quietly.

Not a list. A single line: "Part of a longer story."

Tapping opens the relevant Thread view within Dialogue or the Space.

---

## States

### Normal state
Primary Thought present. Optionally: Focus, Supporting context, Thread indicator.

### Quiet state
No signal strong enough. Today shows minimal content.

Options for quiet state:
- Date only
- Date + "Nothing requires your attention today."
- Date + one line from the most recent active Thread (low-key, not urgent)

Quiet state is **not** an error state. It should feel calm, not empty.

### Warning state
A Warning overrides normal Today content.

Warning takes the Primary Thought slot. Supporting context is suppressed. Focus is the Warning action.

Warning styling is visually distinct but not alarmist.

---

## What Today Never Shows

- A task list
- A calendar view
- Raw metrics (numbers without interpretation)
- More than one primary thought
- Notifications or badges
- "Here are your insights" style digest
- Encouragement or motivational language
- Onboarding prompts after day 1

---

## Caching and Staleness

Today is cached after first assembly.

Cache invalidation:
- New day (midnight) → Today rebuilds
- Warning fires → Today updates immediately
- User explicitly refreshes → Today rebuilds (rare; requires deliberate action)

Today does not silently update during the day. What the user sees at 08:00 is what they see at 14:00.

---

## Relationship to Other Surfaces

**Today → Dialogue:**
The user can tap into any element of Today to continue thinking about it in Dialogue. Today is a starting point, not a destination.

**Today → Spaces:**
Focus and Supporting context items link to the relevant Space. The user can go deeper without Today trying to show that depth.

**Today → Mirror:**
Today never links to Mirror directly. Mirror is pulled, not pushed.

**Today ← Reasoning Engine:**
Today is a display surface. It does not reason. All reasoning happened before Today assembled.

---

## Edge Cases

**First day (no history):**
Today shows a welcome state. AIRCH has nothing to say yet. Quiet state with a short explanation of what Today will become.

**User opens Today after 3+ days of absence:**
Today does not try to summarize what was missed. It shows the current day's signal only. The user can go to Dialogue or Continuity to understand what happened.

**Multiple Warnings simultaneously:**
The highest-priority Warning takes Today. Others are queued in Dialogue for the user to find.

**Primary Thought becomes irrelevant during the day:**
E.g. the event it referenced has passed. Today does not update mid-day. If the Reasoning Engine detects this, it flags the signal as stale for tomorrow's assembly. The user sees the original Today unchanged.
