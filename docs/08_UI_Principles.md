# UI Principles

## Foundation

AIRCH's interface is not a canvas for features.

It is a space for clarity.

Every element on screen has a cost: it consumes attention. The question for every design decision is not "should we add this?" but "does this earn its place?"

---

## Core Principles

### One dominant idea per screen
Every important screen should have one primary thought.

If the user has to decide what to read first, the screen has failed.

AIRCH decides. The user receives.

### Whitespace is functional
Space is not decoration. Space tells the user:

> You do not need to panic. There is room to think.

Density signals urgency. Spaciousness signals calm. AIRCH should feel calm.

### Editorial, not dashboard
AIRCH should feel closer to:
- a thoughtful morning briefing
- a private notebook
- a calm one-page document

than to a SaaS analytics panel.

Dashboard layout implies that all information is equally important. It never is.

### AIRCH speaks first
The interface should not always wait for the user to ask.

AIRCH should orient attention before the user has to look for something.

The first thing a user sees when they open AIRCH should be AIRCH's view — not a blank canvas waiting for commands.

### Metrics become meaning
Do not show raw data. Show what the data means.

Not: HRV 52ms.
But: Your recovery has been falling. Today is not the day to force another PR.

Not: €340 spent on restaurants.
But: Restaurants are absorbing 24% of your spending. That is new.

### Attention before interaction
The first job of a screen is not to offer controls.

The first job is to orient the user's attention.

Controls come second. Clarity comes first.

### Reveal complexity gradually
The first surface should be simple.

Depth appears when the user reaches for it — not before.

A person who wants more should be able to find more. A person who wants calm should find calm.

### Silence is a feature
The empty state is not a failure.

If AIRCH has nothing worth showing, it should show nothing — or near nothing.

A blank moment is a signal: right now, nothing demands your attention.

## Foundation Screens

**Today** — one dominant thought; minimal controls; AIRCH speaks first.

**Dialogue** — clean, continuous, no messenger chrome; thought continues.

**Spaces** — calm overview of a domain; memory and open questions; not a board.

---

## Anti-Patterns

Avoid:
- too many cards competing for attention
- visible raw metrics without interpretation
- notification-style density
- widgets that imply everything is equally urgent
- empty states filled with onboarding copy
- loading states that feel like computation anxiety
- excessive navigation depth
- screens that look like competitors

---

## The Test

After any design decision, ask:

> Does this make the experience feel calmer and clearer — or busier?

Busier means remove it. Or hide it. Or defer it.

Calmer means it belongs.
