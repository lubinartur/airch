# Mirror

## Definition

**Mirror** is what AIRCH believes about the user — made visible and editable.

It is not a profile. Not settings. Not a preference page.

It is a living model of AIRCH's understanding — the beliefs the system holds, the patterns it has noticed, the assumptions it is making.

---

## Core Question

> What does AIRCH believe about me?

---

## Why Mirror Exists

A system that claims to know a person must let that person inspect and correct what it believes.

Otherwise, memory becomes surveillance.

Mirror is the trust contract of AIRCH.

Without Mirror, the user must simply hope that AIRCH's understanding is accurate. With Mirror, they can verify, correct, and guide it.

---

## What Mirror Shows

**Stable facts**
Things AIRCH is confident about: values, preferences, working style, commitments. High confidence. Rarely wrong.

**Observed patterns**
Recurring structures AIRCH has noticed across time and domains. Medium confidence. Shown honestly as patterns, not certainties.

**Current beliefs**
What AIRCH thinks is true about the user right now — their state, priorities, direction. Lower confidence. More provisional.

**Assumptions**
Things AIRCH is acting on that the user has not explicitly confirmed. Surfaced so the user can validate or correct.

**What AIRCH does not know**
Gaps in understanding. Areas where AIRCH has low data or conflicting signals. Shown honestly.

---

## What the User Can Do in Mirror

- **Confirm** — "Yes, this is true."
- **Correct** — "This is outdated / wrong. Here is what is actually true."
- **Delete** — "Remove this entirely."
- **Add context** — "This is true, but here is what you are missing."
- **Adjust weight** — "This matters more / less than you think."

Every correction updates Memory confidence and refines future outputs.

---

## Tone in Mirror

Mirror speaks in AIRCH's voice — direct, honest, without defensiveness.

It presents beliefs as beliefs, not facts:
- "AIRCH believes..." not "You are..."
- "This appears to be a pattern..." not "You always..."
- "Based on the last six months..." not "You tend to..."

Mirror is honest about uncertainty. High-confidence items are stated clearly. Low-confidence items are framed provisionally.

---

## Mirror Is Not

- a user profile (name, photo, preferences)
- account settings
- a memory dump or database viewer
- a list of facts sorted by date
- a surveillance record

Mirror is AIRCH's understanding of who this person is — not a log of what they did.

---

## Mirror and Trust

The existence of Mirror is what makes every other part of AIRCH trustworthy.

When AIRCH makes a recommendation based on something it believes about the user — the user knows they could open Mirror and see whether that belief is accurate.

This changes the relationship from "AI that knows things about you" to "system whose understanding you can verify."

That is the difference between a tool that feels safe and one that feels like surveillance.

---

## Access Pattern

Mirror is not opened daily. It is a destination for when the user wants to understand what AIRCH knows, or when AIRCH has said something that feels off.

AIRCH may occasionally prompt the user to review a specific Mirror item — when a belief has been unconfirmed for a long time, or when new data contradicts something previously assumed.
