# Memory Model

## Purpose

Memory is what allows AIRCH to know a person rather than just respond to them.

Without memory, AIRCH is a chatbot. With memory, it becomes a system that understands.

Memory is not a database. It is a living model of understanding — built gradually, correctable at any time, always in service of better judgment.

---

## Memory Types

**Observation**
A raw signal: something happened, was said, or was detected. Short-lived. Becomes a Fact if confirmed.

**Fact**
A confirmed observation. Stored with source and date. The building block of everything above it.

**Event**
A meaningful moment in the user's life. Has context, domain, and significance.

**Decision**
A choice the user made — or avoided. Especially valuable for understanding patterns over time.

**Preference**
How the user likes to work, live, communicate. Inferred from behavior, confirmed through Dialogue.

**Relationship**
How the user relates to people, projects, and domains over time.

**Habit**
A recurring behavior — positive or negative. Detected from patterns across Events.

**Pattern**
A repeated structure across time or domains. The first level of real understanding.

**Value**
What actually matters to the user — not what they say matters, but what the data shows. The deepest layer.

---

## Memory Lifecycle

```
Observation
↓ (confirmed by recurrence or user acknowledgment)
Stable Fact
↓ (repeated across contexts)
Pattern
↓ (connected to values and direction)
Long-term Understanding
```

Memory at the top of this stack feeds the Brain. Memory at the bottom is raw material.

Not everything becomes Long-term Understanding. Most observations stay at the Observation or Fact level and decay naturally if never reinforced.

---

## Memory Confidence

Every memory carries a confidence level:

- **Observed once** — low confidence. Stored but not acted on.
- **Observed repeatedly** — medium confidence. May surface as a question.
- **Confirmed by user** — high confidence. Used in recommendations.
- **Rejected by user** — confidence drops. May be removed.

AIRCH never presents low-confidence memory as fact.

---

## Memory and Mirror

All memory that rises to the Pattern or Understanding level is visible in Mirror.

The user can:
- Confirm that something is true
- Mark something as outdated
- Correct a misunderstanding
- Delete something entirely
- Add context AIRCH missed

Memory must be editable through Mirror. Otherwise it becomes surveillance.

---

## What Memory Should Never Feel Like

- A database being queried
- A log of everything that ever happened
- A surveillance record
- A list of facts without meaning

Memory should feel like a person who has been paying attention — and who uses what they know naturally, without announcing it.

Not: "According to my records, you mentioned this on March 14."

But: "We have been here before."
